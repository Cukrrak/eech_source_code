What's NEW in EECH 1.9?

(since version 1.8.6)

Weapons modelling
  • Gun projectiles and unguided rockets are now affected by gravity, drag and dispersion. As a result rounds take longer to reach their target and with less accuracy.
  • Vikhr missiles now follow a spiral flight path
  • Vikhr launch racks will depress up to 12 degrees to align with target direction. This helps as the Vikhr now has very narrow launch restrictions for weapon launch.
  • Rockets and missiles will now only create smoke for as long as the rocket engine is burning.
  • Rockets adjusted to have shorter burn time and faster acceleration.
  • Muzzle velocity adjusted for many weapons to better match real world values.
  • New values for GWUT file for changing weapons drag, spiral path and seeker limits. These are included in gwut190.csv, but note that the GWUT editor (version 1.3) is not yet capable of editing these values, and will delete them from any file which contains them. So you will have to edit the file with a text editor until a new version of the GWUT editor comes out
Avionics changes
  • New weapons aiming avionics for ballistic weapons. See wiki article for details.
  • New flight MFD page, revised TADS page and added colour to many other MFD pages for Comanche.
  • New HUD for Comanche
  • New flight, TSD, radar and engine pages for Alligator. Added colour to several other pages.
  • DTV/LLLTV will now show a full-colour(for Comanche) or monochrome (for the others) TV view, not a IR-view.
  • DTV/LLLTV in Comanche and Ka-52 is now capable changing zoom level.
  • EO zoom level can now be changed even if EO is not active targeting system. Use CTRL + keypad +/- for this.
  • EO camera can now be changed even if EO is not active targeting system. Use Shift + END for this.
Campaign map changes
  • New relief shaded topographical campaign map
  • Hi-res campaign map and alternative map palette also available (campaign_map_mode and campaign_map_palette in eech.ini)
  • Adjustable how often enemy units update. By default the position of known enemy units are updated once every 2 minutes. Configurable by map_update_interval in eech.ini.
  • Fixed bug where undiscovered enemy units would show up if you just zoomed the map enough.
  • Campaign map in free flight mode now allows adding and moving waypoints.
External cameras
  • New free camera mode. Press F9 when already in chase or weapons camera view. Use mouse to pan, arrow keys or mouse wheel to move camera.
  • New wobbly camera option (wobbly-camera in eech.ini) for chase and weapons camera. Gives the impression of the camera being mounted in a chase plane/helicopter which is not always able to keep perfect view of object.
  • Chase camera allows a limited amount of movement from object being chased. Use arrow keys for this.
Smoke fixes
  • You can now disable the persistent smoke (persistent_smoke=0 in eech.ini). With it enabled smoke will only be created for about 10 seconds, instead of minutes.
  • Rockets and missiles only generate smoke as long as the rocket engine is burning.
  • Slightly adjusted smoke colour.
  • Fixed a crash which would sometimes happen with smoke.
Other changes
  • Fixed a problem for multiplayer/saved games where data would get corrupted
  • Trim does no longer affect helicopter attitude in hover hold mode.
  • ... and probably some more changes I've forgotten to mention, and numerous bugs I haven't discovered yet :)

Installation and addons

NEW: Go to www.eechcentral.com and download the installation tool: eechupdater1.3.exe

This will download and install all patches and addon packs that you need. If new packs arrive this tool will detect them and download them, keeping your database up to date. It is still a good idea to read the following!

Basic Install

If you do not own EE - Apache Havoc you can skip the first two steps

  1. Install EE - Apache Havoc from CD and run once to unpack the graphics files.
  2. Patch EEAH up to patch 1.1E, run once.
  3. Install EE - Comanche Hokum from the CD.
  4. Do not forget this: Download and install patch 141C from the Razorworks site: documentation is here

If there are new options the old eech.ini will still be read and the new options added when you leave the game. Start the game once and leave it to see which options are added.

IMPORTANT: when you have just installed eech by default the avionics in the options menu is set at "novice". This means that you cannot operate the targetting systems, it is done for you in the game by the "copilot". Set this option at advanced when in the game for full operation. When you delete the game.cfg and it is recreated by the game, the option is reset at novice!

Trouble shooting

If your game doesn't run properly, there is not so much that can be done!

  • If you have some weird behaviour, go to the directory razorworks\cohokum and delete the files game.cfg which contains all the settings of the in-game user interface, and eech.ini which contains additional and settings for mods, in the cohokum directory. These files are regenerated automatically. NOTE: when you delete game.cfg the avionics in the options screen is reset to "novice" and the targeting systems do not work, you have to set it to "advanced" to make it work again.
  • A fresh install has been known to help in many cases (but its a pain).
  • Set the screen refresh to "always on" for Radeon cards.
  • Try in the options screen to set the Device selection to either automatic or Primary Display Driver. This may sometimes help.
  • Graphics settings in eech.ini: try changing the notnl setting
  • Some people reported a problem with NVIDIA drivers 71.xx, no cure yet.
  • Very old saved games may not work because certain modded settings will not be in the game. This is rare however.
  • Framerates: these cannot be improved much because of the old rendering system. Framerate killers are FLIR especially above cities. Also in the eech.ini settings, the options "highresMFD", " high_lod_hack" and "cbar" should be set to off or a low value. Set mfr to a high value (999). Note that the best way to test framerates is with the tool "FRAPS". Of course on your graphics card AA settings may have a negative effect (but not always, this is not the bottleneck), and the "texture preference" can be set to "high performance" instead of "high quality".
  • If you have windows XP and can't run apache havoc try running it in win 98 mode: right click on the executable and its a setting in properties -> compatibility.
  • Please try to repeat the error and report about it on the SimHQ - EECH forum. There is a file error.log in your cohokum directory when the game crashes but it is a memory stack dump and pretty useless, no need to report that.

Game info

There are a number of ways to play this game:

  1. As is, default settings, default graphics. Great game.
  2. Download the superpatch2 with tons of custom made warzones/campaigns/skirmishes
  3. Use a WUT file to change the behaviour of vehicles and aircraft all in the game , increased difficulty (some say more realistic).
  4. Use custom textures to improve the looks of the game.
  5. Browse through the eech.ini and activate for instance "faa=1" (fly any aircraft, jump in returning choppers and have a field day) or for instance "camcom=1", the campaign commander which lets you create missions and give you a greater control of the campaign.
  6. Play online with MP (look on this server to see what's up).

Flyable choppers

As of version 1.6.0.W12 you can fly, change the loadout and repair the following choppers: Comanche, Apache Longbow, HokumB, Havoc, Blackhawk, Hind, AH64-A (Apache), and KA50 (Hokum). The cockpits and the flight behaviour of the new blue choppers default to the Apache Longbow, of the red choppers to the Havok.

Note that you can pick a returning chopper to fly rogue. Spot the wanted Helo in the 2D-map, click on it, accept, auto-select (mission), confirm and off you go (you can't choose it from the mission or group list since it has no task). You can only do this if the option "faa" in eech.ini is set to "1" (see eech.ini).

Jabberwock made any chopper flyable in free-flight mode. Just be sure you activate the "fly any aircraft" switch in eech.ini. Use F12-key to see the aircraft and scroll through the list to the chopper you want to use. Hit F12 again to clear the screen and then hit 'U' and you are in the cockpit of the chosen chopper.

Multiplayer status

Seems to be fairly smooth these days, but the test environment is limited, so there could still be bugs left. It's crucial that the host runs with cvc=1. Its also a good idea to touch only the other variables if you know what you are doing. Downwash should be off, it can mess up the host after a while.

What is a WUT file?

Most people fly with a WUT file. A WUT-profile (Weapons and Units Tweaker file) is a text file that alters some critical parameters in the game to make it more "realistic" and harder. The text file name can be added to the file eech.ini and all parameters are read by cohokum.exe at startup. The most used WUT file is gwut151D.csv which basically increases speed and scan capabilities of aircraft and vehicles and power of various weapons (increasing gameplay difficulty). Furthermore there is gwut1613H.csv which adds enemy chopper aggressiveness (all enemy choppers behave as if they are on a strike mission), and gwut146X.csv which is needed when playing multiplayer and has the game's default values (identical to playing without a WUT file). A last GWUT files has bee added: gwut162B.csv by KLiM, see here for a description.

You can edit these files with the GWUT editor, downloadable from Gotcha's site. If you intend to join a MP-game all three WUT files are needed. You are not aware of the host's profile until you're connecting and if the host is running with one of the WUT profiles your game will look for a similar one in your cohokum-folder.

The eech.ini file and mod info

These are only the modded variables! In order of appearance in eech.ini:

[Communications]

Look for master server

usemaster=0

When starting the game it reports to the master server and gets info from it (value = 1), when set to 0 the master server is ignored (default = 0)

Multiplayer master server

pss=[server name]

sss=[server name]

Primary and secondary server setting. At http://hoxdna.org/eech there is a master server for EECH, listing all running games along with total players and IP addresses. EECH versions newer than 1.6.0W1 report to the master server if you play a game with Networking set to TCP/IP in the options menu. To join a game, just copy&paste an active IP from the server list into GSS (Ground Support System) or EECH itself.

Server pause when client connects

pauseserv=1

When a host starts a game, he/she can hit "pause" and the game will resume when a client connects. When the (last) client leaves the game returns to "pause"-state and resumes again when a client connects (default=0, off).

[Dynamics]

Advanced flight model

advancedfm=0

If enabled, this setting switches to a more complicated flight model. Pedal mixing will be disabled, which means that you will have to use the pedals to turn the chopper. This makes it possible to fly sideways to track and attack moving vehicles, for example.(default=0, off) .

Realistic engine dynamics

enginerealism=1

Causes the game to simulate the workload that the engines and the rotorhead are stressed with, and causes engine RPM to drop under prolonged extreme stress. Also, engine torque INCREASES with increased workload. This setting also makes the engines sound more lifelike. (default=1, on) .

enginestartup=0

If enabled will begin mission with engines off, and the pilot will have to start the engines as per engine start up procedures (default=0, off)

Debuging

debug_show_force_vectors=0

Show force vectors acting on own helicopter. This is mostly for debuging the flight model (default=0, off)

[WUT]

wut=[filename]

WUT files are text files with Weapons and Units Tweaks to make the game harder. As of version 160W15 the new version "GWUT" files can be read as well as old style WUT files. For more information visit Gotcha's site.

[wideview]

Change your point of view to see more of the cockpit

commanche_pilot=x,y,z,p
commanche_co-pilot=x,y,z,p
hokum_pilot=x,y,z,p
hokum_co-pilot=x,y,z,p
apache_pilot=x,y,z,p
havoc_pilot=x,y,z,p

Retreating the pilots head causes the player to have a wider field of view without distortion. The values can be edited by pressing Ctrl-\ in the game and using the numpad keys to move the viewpoint: 4 and 6 move distance to dashboard; 2 and 8 move vertically; 1 and 3 move horizontally, 7 and 9 change the pitch (set free mouselook off). Ctrl-\ quits editing, when leaving the game the values are saved in the eech.ini file.

Note: for some keyboards with strange and disturbing langauage settings the keys are not \ and Ctrl-\ but something else. You just have to try for yourself on the right hand side of the keyboard. For more info: Gotcha's site

g-force_head_movement

Adjust amount head will move with g-forces in wideview. Default = 1.0, off = 0.0 all other values adjust amount of movement (may be set to more than 1.0 or even negative values to move head in opposite direction)

Controllers & Track-IR

Mouselook and TrackIR support

msl=1
msls=15

You can now control you view using the mouse or by using TrackIR. The TrackIR support uses enhanced mode which means that you will not have to bother about centering you view all the time. A check will be made for a TrackIR upon startup, and if it is found it will be used to control the view. Otherwise the mouse is used. (default = 0, off). Pressing the left mouse button will recenter the view to forward.
If mouselook is on, the msls setting controls the speed of the mouselook (6-20, default 15).

Mouse controlled FLIR

msl=0
msls=15

When mouselook is off (msl=0), you can use mouse for FLIR panning. Additionally, when you press left mouse button you lock/unlock target, with right button you can select next target and with mouse wheel you control zoom level.
It is also possible to switch mouselook on/off with ctrl-delete key combination or with a middle mouse button.
If mouselook is off, the msls setting controls the POV rotation speed (6-20, default 15).

Field of view

minfov=20
maxfov=80

The user can specify different field-of-view values, which can be activated by the '7','8' and '9' keys. 7 switches min FOV, 8 switches to (fixed) standard FOV, 9 switched to wide FOV. Please note that higher FOVs currently mean lower LODs.

Joystick controlled FLIR panning and zoom (Comanche and Hokum only)

eopann=-1
eopanv=6
eopanh=8

eozoomn=-1
eozoomax=7

You can now use joystick axis to pan the FLIR display. You specify the joystick number and which axis to use for this feature.
The old FLIR code only allowed three levels of zoom. This code has been updated to support any zoom levels. When using the keyboard, you still use the same keys to zoom, but you now have 5 levels instead. If you have an extra joystick axis available you can map one of them to the zoom and using for zooming.
DirectX numbers, general names, Saitek X36USB/X45 names:
1 - X-axis -  X-axis
2 - Y-axis -  Y-axis
3 - Z-axis
4 - rx-axis -  Rotary 2
5 - ry-axis
6 - rz-axis  - Rudder
7 - slider 1 - Throttle
8 - slider 2 - Rotary1

Mappable joystick axes

cyclicn=-1
cyclich=1
cyclicv=2

collectiven=-1
collectiveax=3

ruddern=-1
rudderax=6

You can now choose exactly which joystick and which axis on that joystick that should be used for cyclic, collective and throttle. See "FLIR panning" for DX numbers.

Gameplay

radarinf=1

infantry no longer visible on radar, def = 1 (on)

grstab=1

The Comanche now supports ground stabilisation for the FLIR. You toggle this feature in-flight using Ctrl-S.

manual_laser/radar=0

Radar and laser activation/deactivation is manual, and not controlled by targeting mode, def = 0 (off)

faa=1

Fly any aircraft: all helicopters available now for the user. Fly your own insertions! The cockpit defaults to the Apache when choosing anything else than CHAH (default = on) .

Graphics and textures

dwash=1

Rotor downwash (dust) effect. It makes you see clouds of dust when the chopper gets close to the ground (default=1, on) . May give a framerate hit when activated.

restricted_nvg_fov=1

When enabled the night vision will show a dark frame, severly restricting the field of view for a more realistic night vision effect.

dfr=0

The framerate is displayed in the top left corner and updates about once a second. The average of 30 readings is shown for stability. Values: 0 = off; 1 = on; 2 write the framerate to a file framerate.txt in the cohokum directory. This is not a very good measure, you better use the FRAPS tool.

highreshud=1

The resolution of the HUD will be doubled. Currently only the Ka-52 and Mi-28 has a high res mode for the HUD, for the other helicopters this setting has no effect.

colourmfd=1

MFDs will use colours for display when available. (But the TSD map will always be in colour).

Special TSD render

tsdrender=1
tsdpalette=2
tsdenemy=0
tsddetail=0

This mod adds a color and/or shaded relief map background to the TSD of the Comanche, Hokum and Apache (the Havoc is not included). Ctrl-D cycles through the options. There are 3 palettes to choose from for the elevation (Alt-D). The Apache has the same TSD as the Comanche.The configuration is automatically saved in the eech.ini file when you leave the game.
Optional: the friendlies and enemies are shown in red and blue colours (including detection circles of SAMs). If you don't want this option set the eech.ini variable "tsdenemy" to 0. There is may be a framerate hit when showing the terrain. If tsddetail is set to 1 the TSD map will always use high detail level map, even if in 25KM range mode (might cause lower frame rate). For more info:Gotcha's site .
Campaign commander

camcom=0

When the camcom setting is set to 1 (def = 0), it is possible to designate targets on the main campaign map. Just select the enemy target and press left mouse button: a new task is created. New tasks work the same as other tasks: they may not be executed right away, may expire etc. When you press right mouse button over a task, it is immediately terminated. If it was assigned to any group, the group is sent home. Note that Campaign Commander greatly influences the gameplay balance: with the right planning the campaigns are much easier to win.

Designated target list

destgt=0

Actvate target list (default =0). Make a list of targets to attack: Alt-NumEnter to add/remove targets to the list. You can select a special MFD filter (added to the low, medium, high that already exist), where you can select the targets with the regular keys. There are also two keypresses: Alt-Num0 and Ctrl-Num0, which allow to select designated targets even outside the filter (default =0, off).

Session filtering

filter=1

You can filter the campaign screen to sort the campaign names according to their warzones. (def = 1)

Pedal input

reverse_pedal=0

reversed pedal input
High LOD
high_lod_hack=0 EXPERIMENTAL! Enables highest level-of-detail models at far distances. Nice for higher FOVs, bad for FPS (esp. near cities). Default = 0
3D apache cockpit
3d_cockpit=0 EXPERIMENTAL! Draws a 3D apache cockpit using an in game apache model. Very ugly but fun. Default = 0;

TrackIR external view

external_trackir=0
external_trackir_dir=0		 if TrackIR is active, let it control external view too (def=0). Invert the view direction of the external TIR (def=0)
Cannon tracking boresight
cannontrack=1 Cannon is tracking the pilot's view. Three settings are available: 0 = tracking off, 1 = tracking only when no acquisition system is turned off (Ctrl-NumDel), 2 = tracking only when IHADSS/HIDSS/HMS system is active.
TrackIR vector support
TIR_6DOF=0 Enables support for TrackIR vector
Texture support

texture_colour=0
texture_filtering=0
mipmapping=0
dynamic_water=0

Set these values to 1 to enjoy high res mipmapped alpha blended custom made texture packs. When linear filtering is set to 1 the game reacts to the anisotropy setting of your graphics card. Dynamic_water simulates flowing water but only with the correct texture packs.
Autosave
autosave=0

autosave=n: Autosave every n minutes or 0 for no autosave. Quicksave is also possible using Ctrl-Z.

Version history

Fixes/features in 1.8.6:

  • Fix for save games and multiplayer crashing in 1.8.5.
  • Export also the upfront display/ekran data to shared memory (fix by Retro).

Fixes/features in 1.8.5:

  • Added arrow keys as fine adjustment of trim. Useful in when helicopter is mostly trimmed, but just need some fine adjustment. Only works when using a joystick for cyclic control.
  • Added shared memory export of flight data (thanks to Retro)
  • Added waypoint, weapon and NG RPM data to shared memory export.
  • Several crash fixes including when taking damage in Cobra and other helicopters with general modelling (thanks to Richard Evans for all the crash fixes).

Fixes/features in 1.8.4:

  • Fixed Hind's flight model which caused it to be suddently do violent manouvers and crash.
  • Reverted fix which caused some missiles to continue unguided when they lost guidance. They will now self destruct again. The unguided fix caused crashes in multiplayer.
  • Fixed so that mouse buttons can be used with the EO cameras again.
  • If the registry doesn't contain a Apache Havoc entry, EECH will now check if the aphavoc directory exists in the same directory as cohokum, and use it if it does (only used for Apache and Havoc cockpit textures).

Fixes/features in 1.8.3:

  • Fixed bug where save games would come up as invalid even if saved with same version.
  • Reverted Gotcha's camo fix as it caused more problems than it solved.

Fixes/features in 1.8.2:

  • Several changes to guided missile flight model
    • Missiles might now lose guidance if target moves outside seeker's limit. Air-to-air missile will self destruct if this happens, air-to-ground missiles will continue on unguided.
    • Hellfires launched in LOBL mode should now be able to hit targets at close and medium ranges again :)
    • Hellfires launched in LOAL mode now always does the initial climb and hence won't be able to hit targets at close range (closer than 2km).
    • Seeker field of view reduced for the Russian Anti-tank missiles (Ataka, Vikhr, Spiral). Used to be around 25 degrees from center, now is 10-15 degrees (if using the new gwut file).
    • Anti tank missiles and other missiles with shaped charges will do less collateral damage now, as most of the blast is concentrated on the target hit directly.
    • Air-to-ground missiles which lose their lock will not self destruct but continue on (until hitting the ground, getting too slow, or if a semi-active missile until it has a new target inside seeker limits).
  • Several Additions to the pilot log
    • New option to restore log to what it was before last mission (i.e. erase last mission). Useful if e.g. the game bugs causing you do die. Only works if you do it right after the mission, if you exit EECH you won't be able to erase the mission the next time you start.
    • New page with flying hours and number of missions for each helicopter type.
    • More detailed statistics on which types of targets have been destroyed by the pilot.
    • New statistic for number of helicopters lost. This is the number of helicopters crashed, or landed outside of a base. For converted logs this will start with the same number as KIA.
    • KIA value will now be increased if you die also before the mission is completed.
    • A mission in which you crash is now counted as a mission flown, even if mission was not over at the time of the crash. You will however not get any points (used for calculating promotions and medals).
    • As a result of these changes this version uses a new format for the pilot log, and a new file (player2.bin). Old logs will be converted to the new format on first start of 1.8.2. But the unknown values (the ones which were not stored previously) will be 0, except for helicopter kills which gets set to the number of air kills the old log had (even though some of those kills might have been fixed wing aircraft).
    • Fixed bug where some missions were logged twice (and not necessarily with the same result).
  • MFD changes
    • When using manual startup, the helicopter will now start with HUD system and MFDs powered down (except for the co-pilot's, he has already activated his own). Use Ctrl-K to active HUD, and Ctrl-[ and ] to active the MFDs.
    • ALT [ and ] will now not only activate and deactive the Comanches side MFDs, but also cycle their mode.
    • Apache's TADS and radar MFD pages now has time until impact for hellfires in the high action display area. And in TADS a flashing L has been added below and to the right of the target gates when the laser is active.
    • Fixed overlaid MFDs being too dark with night vision in pilot view without cockpit.
    • Fixed some of the Longbow's new MFD modes which didn't work in overlaid mode.
    • Fixed TSD's mouse pointer which didn't work in overlaid mode.
    • Increased size of overlaid MFDs a little for Longbow and Havoc, to make them easier to read.
    • Overlaid MFDs now always use big size, no matter what HUD size you're using.
    • New eech option (targeting_system_auto_page) to control if changing targeting system will also automaticly switch to the targeting system's MFD page.
  • Other changes
    • Implemented non-linear axis for cyclic, collective and pedals. Cyclic and pedals are less sensitive around centre, making fine adjustments easier. Collective can be adjusted to have bigger range around 60-100%, where most of the time is spent. Makes hovering easier with better control and finer adjustments.
    • The laser will deactivate when target moves outside of TADS's field of view (or similar targeting system). It will also lose lock on target if target or lasing helicopter makes too violent manouvers for the targeting system to follow. The laser will most likely also disengage when switching targets in manual laser mode.
    • AI wingmen won't any longer need to climb to get line of sight to target when firing radar guided hellfires. They will instead use LOAL mode and fire from behind cover.
    • AI will no longer waste guided air-to-surface missiles. They will not refire a guided missile at the same target until the first missile has missed (or hit but failed to destroy target). Not just fire a new missile every few seconds even if they have several missiles heading for the target already.
    • Fixed crash with A-model Apache, and all other helicopters using its cockpit.
    • Fixed bug which caused trying to throttle engine down before it was started to actually start the engine.
    • There was a conflict of keys in last build. Both deselect PFZ and deactivate targeting system was on CTRL NUMPAD DEL. So now things have moved a little. Deactive targeting system has moved to SHIFT-DEL (normal DEL). Should hopefully be a little more logical to keep all select targeting system commands in that keygroup. And now all the PFZ commands are at NUMPAD DEL. ALT deselects current PFZ, and CTRL delets it (and SHIFT and plain cycle PFZs like it used to do).
    • Changed "use master server" eech.ini option to default to off. That way you won't be broadscasting your IP (and other info) just by enabeling multiplayer.
    • Fixed bug with desert camo (gotcha)

Fixes/features in 1.8.1:

  • Longbow's threat warning MFD page (ASE) has got colours. SAM sites have RED rings, AAA yellow. IR missiles are red, radar missiles are yellow, and laser guided missiles are blue.
  • Longbow's ASE has threat identification. Below the threat symbol is a number identifying the type (19 for a SA-19, 29 for a MiG-29 etc. The only non obvious id is N which stands for naval, i.e. the air defence system of a ship).

It's not perfect yet, if the helicopter is not available in the selected campaign it will crash (although all helicopters are available in most campaigns). Also the 3D model for the other helicopters are a little weird with too many subobjects.
  • Fixed crash bug in Comanche when using manual laser
  • Fixed background colour in some Longbow MFD pages appearing too bright at dusk and dawn
  • Fixed mixup of greenmfd and colourmfd in eech.ini
  • Fixed position of hellfire time of flight for US HUDs
  • Fixed that Longbow weapons MFD page would list laser as range finding system even when laser was off. It now displays TRNGL (for triangulation) when laser is off.

Fixes/features in 1.8.0:

  • Avionics changes
    1. Added option for manual laser and radar control. With it enabled you will have to activate and deactivate the laser and radar manually (keypad * and / keys respectively. Switching radar to single sweep mode has been moved to SHIFT + keypad /). In non-radar targeting modes the laser needs to be active to get accurate range to target, and must be active to fire laser guided missiles. Radar activates automatically when switching to a radar targeting mode with the option off. With it on you can activate/deactivate the radar at any time regardless of which targeting mode is active.
    2. Removed radar as targeting system from all helicopters except AH-64D, RAH-66, Mi-28N and Ka-52.
    3. Removed TADS as targeting system from Blackhawk.
    4. Added new MFD page for Apache: flight, which displays some of the same information as the IHADSS.
    5. Added different underlays for the Apache TSD page; None, map and TADS imagery. Switch between the three with SHIFT+D.
    6. Added Priority Fire Zones (PFZs) and No Fire Zones (NFZs) to Apache TSD. See documentation
    7. Added zoom capability to the Longbow's radar. Press SHIFT-KEYPAD- to zoom in on selected target, SHIFT-KEYPAD+ to zoom out to full view again. In zoomed view the radar view shows targets in a 500x500 meter area around the selected target.
    8. Updated Apache TADS and radar pages to show more information
    9. Added colours for some of the Apache Longbow MFD pages (TSD, flight page)
    10. Longbow's cockpit has gotten full head position movement.
    11. Disabled low-res mode for the MFDs as there is practically no frame rate difference, and no GFX cards out today which don't support 256x256 textures.
    12. Added option for selecting detail level of TSD map - tsddetail in eech.ini (gotcha).
  • Weapons changes
    1. Changed Hellfire flight profiles. LOBL Hellfires now follow a slightly high trajectory (instead of going straigt for target), LOAL Hellfires follow a high trajectory.
    2. If you have a target selected already when firing in LOAL mode the Hellfire will go for that target, even if you switch target before it finishes its climb. This makes riple firing Hellfires in LOAL much easier (particulary in combination with PFZs).
    3. Laser must be active for laser guided missiles (K Hellfires and Vikhrs) to guide. Laser is acticated automatically unless the manual laser option is enabled in eech.ini.
  • Night changes
    1. Added eech.ini option for adjusting night level brightness (thanks to gotcha).
    2. Changes colour of russian night vision from amber to a blueish-green colour.
    3. Added optional shading of screen edges (restricted_nvg_fov in eech.ini) with night vision active for a more realistic restricted field of view.
  • Other changes
    1. Added eech.ini option for showing force vector on own helicopter. This is mostly for debugging the flight model, or if you're just curious which forces are acting on the helicopter.
    2. Added new radio message to flight which requests status. Wingmen will reply with how many weapons and how much damage they have.
    3. Added new radio command for transfering PFZ with targets to wingman and ordering him to attack all targets in it.
    4. Allow each campaign/skirmish to have its own formscomp.dat file. Will use file in common/maps/mapx or common/maps/mapx/campaignname if they exist. This allows each campaign/skirmish to have its own force composition.
    5. Increased default initial multiplayer buffer sizes for hopefully less connection problems in multiplayer (sugggested by Kimchoc1).
  • Bug fixes
    1. Fixed the head-on-a-stick movement of the head in wideview.
    2. Fixed a bug in the Hokum and Havoc flight model where it would suddenly start doing violent yaw movements, and eventuelly go out of control.
    3. Fixed bug where UI would sometimes freeze in session screen if multiplayer master server was down.
    4. Fixed crash bug in Hokum's HUD view.
    5. Changed displayed name of Hind from Mi-24D to Mi-24V, as the model simulated in EECH is clearly not a D, and appears to be a V.

Fixes/features in 1.7:

Flight model changes
  1. Numerous changes to make autorotation work (read here to learn how to autorotate).
    • Rotor brake is now actually engaging/disenging rotor to engine, it's not a brake
    • The collective now works also when the rotor is disengaged
    • Landing with engine failures don't automaticly make the helicopter explode
    • Created new aerodynamic model for rotor RPM when rotor is disengaged. Rotor RPM is now affect by autorotation which increases rotor RPM, and induced and profile drag which reduces rotor RPM.
    • Rotor makes no torque when rotor is disengaged
    • When the rotor is disengaged, the main rotor drives the tail rotor so that it doesn't stop.
    • When the rotor is disengaged rotor rpm may exceed 100%, but get it too high and the rotor might fail as well. Add collective to slow the rotor down again.
    • Rotor automatically disengages if both engines fail.
  2. Numerous changes to make engines more realistic. Read here to learn how the engines work now.
    • Added modelling of gas turbine part of engine (and not only power turbine, as it used to be)
    • Engine control system automaticly controls gas turbine RPM in an effort to keep rotor RPM at 100%
    • Added abillity to control throttle and starters for engines, including full startup procedure
    • Added option for starting mission with engines off
    • Modelling of APU added
    • Modelling of engine temperature added
    • Engine fire due to overtemperature now happens at random time, probability depending on temperature.
    • Improved engine instruments to show engine information which previously was not modelled.
    • Modelled sounds for turbines and APU to correspond with RPM of turbines
    • Fuel consumption depends on engine RPM
  3. Slow speed flight model tweaks. Should make slow speed handling easier and more realistic, particularly sideways flight.
  4. Decreased effectivness of wheel brake
  5. Implemented rotor vibration. Used when close to entering vortex ring effect, or when main rotor slightly damaged.
Avionics changes
  1. Completely new HUD for the ka-52, based on the real Hokum HUD. Read here for instructions. This HUD is also available in high resolution.
  2. New HUD mode for Apache and Comanche: transition mode. Use Shift+O to switch between the cruise mode (the existing navigation mode) and transition mode.
  3. Changes to existing HUDs. For Apache and Comanche more symbology is common for weapons and navigation mode. A slip indicator has been added to all modes. For Russian helicopters vertical speed scale is visble also in weapons mode.
  4. Implemented high-res MFDs for the Apache and Havoc
  5. Made new weapons page MFD for Apache which is more like the page in the real AH-64D
Sound changes
  1. New engine sound comprised of three components which change during flight according to your flight maneuvers. (Rotor-slapping!)
  2. Sound modifications no longer require modifying sound.dat, just copy the new sound files into the audio directory!
  3. Closing canopy doors reduce sound volume of external sounds
Other changes
  1. Increased speed of main rotor animation
  2. Added key for manually opening and closing canopy doors (Comanche and Hokum only)
  3. New, more immersive UI screens
  4. Make F1 work as center view in mouse look (for mouse only, TIR works as before)
  5. Implemented collimation effect for HUD (the focus on infinity, HUD image moves with the head effect). Currently Hokum only.
  6. Implemented head movement with G-forces in wideview mode (Comanche and Hokum only).
  7. Allow loading of modified 3D models from COHOKUM\3DDATA\OBJECTS directory. Use Firebird's 3D model importer and exporter to edit 3D objects.
